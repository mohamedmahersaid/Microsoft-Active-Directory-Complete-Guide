# Contributing

Thank you for contributing. Please:

1. Read the README and SECURITY.md.
2. Fork this repository and create a branch for your feature/bugfix.
3. Follow PowerShell module structure for scripts in /scripts/modules.
4. Add or update Pester tests under /tests.
5. Run `Invoke-ScriptAnalyzer` and `Invoke-Pester` locally before submitting a PR.
6. Use semantic commit messages and update CHANGELOG.md for notable changes.

Pull request checklist
- [ ] Tests added/updated
- [ ] Documentation updated
- [ ] No secrets included
- [ ] Follows code style (PSScriptAnalyzer rules)
